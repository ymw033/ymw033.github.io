使用hexo创建自己的第一个博客

每当看到大佬们都拥有自己的炫酷好看的博客，着实令人羡慕
如果你也想拥有同款博客，那么Hexo你值得拥有。

这篇博客将简要记录一下在创建我的第一篇博客的过程和遇到的问题，希望对看到这篇文章的俊男靓女们有所帮助。

使用Hexo的前置步骤是需要安装Git和Node.js,由于我很久以前已经安装过了，所以看其他大佬的教程时直接跳过了这两步。

Git的安装

Windows环境下可以直接下载Git并安装
Linux环境下可以使用sudo apt -get install git-core或sudo yum install git-core

#Node.js的安装
Node.js可以直接使用官方的安装程序，可使用https://nodejs.cn/download/该网址

Hexo的安装

1、安装

运行命令：npm install -g hexo
安装成功后可以查看版本：hexo -v

安装好以后，就可以创建Hexo文件夹，在指定路径创建文件夹，右键选择Git Bash Here

输入hexo init
cd
npm install

初始化后，输入hexo n “文件名”就可以创建新的文章，在source_posts\下就能看到”文件名.md”这个文件
（我们可以通过MarkDown编辑器编辑文章内容）

输入hexo server命令，就可以启动本地服务器，默认情况下可以直接使用http://localhost:4000/在浏览器直接访问。

2、更换主题
这不是一步非必须的步骤，但是默认的主题实在太朴实了哈哈哈，当然是想让自己的博客好看一点。
我使用的主题是Fluid
在themes文件夹下打开Git Bash Here，使用
git clone git@github.com:fluid-dev/hexo-theme-fluid.git
克隆后将文件夹重命名为fluid并在根目录_config.yml文件更改主题名为fluid
然后重启Hexo，使用hexo clean和hexo g进行重启。

3、部署GitHub Pages
这一步是能够让其他人能够欣赏你的博客的关键。
首先你需要有自己的GitHub账号，其次需要创建一个公有的仓库。仓库名称一定要是“用户名.github.io”这种格式。
可以通过Https或者SSH将文章上传到GitHub仓库，本人一开始选择的是HTTPs的方式，但是后来遇到好多问题，最后还是使用的SSH的方式。

git config –global user.name “GitHub 用户名”
git config –global user.email “GitHub 邮箱”

创建SSH密钥：ssh-keygen -t rsa -C “GitHub 邮箱”
在C:\Users\用户名.ssh目录会生成id_rsa.pub文件，我们可以复制里面的内容到Github上，Setting->SSH and GPG keys新建SSH密钥，将id_rsa.pub文件里的内容复制到里面。

在GitHub配置好SSH后输入ssh -T git@github.com进行连接验证。
我遇到过一个问题困扰了很久，就是编辑好的文章进行提交Github时一直提交不上去。后来进行SSH连接验证时发现，报错ssh: connect to host github.com port 22: Connection refused，经排查后发现这个错误是因为连接github.com的22端口被拒绝了，解决方案是更换为443端口。

输入命令vim ~/.ssh/config
将Host github.com
Hostname ssh.github.com
Port 443
添加到config文件中。

更换端口后，再进行检测连接就会发现连接成功。这个坑整整困扰了我一天，就是提交不上，在这里希望能帮助大家避免这个问题。
哦，对了，如果你疑惑为什么.ssh/目录下为什么没有config文件，恭喜你，你发现了华点。因为就是需要自己新建，哈哈哈。
